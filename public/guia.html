NexVoz

Guía para dejarlo funcionando al 100%

Recepción con IA 24/7 + sitios web que convierten · Modelo agencia → empresas cliente

Tu modelo: tú eres la agencia (NexVoz) y le instalas/cobras el servicio a empresas cliente (restaurantes, comida rápida, farmacias, clínicas, oficios…). Lo único que requiere tus cuentas con tarjeta es el agente de voz (Vapi + Twilio); todo lo demás ya funciona.

Cómo funciona (la idea)

Cliente llama por teléfono → Número (Twilio) → Agente de voz IA (Vapi) conversa y toma los datos → Envía los datos a tu servidor (este proyecto) → Se guarda y avisa al WhatsApp del dueño del negocio → Todo queda visible en el Panel (/dashboard)

Fase 0 · Lo que necesitas

  • Computador con internet y Node.js 18+ (nodejs.org).
  • Cuentas: Vapi (voz), Twilio (teléfono), OpenAI (IA), Render (publicar). Opcional: ElevenLabs (voz).

Costos aprox: voz ~USD 0,08–0,15/min (~$80–$150 CLP/min) · número Twilio ~USD 1–2/mes · Render gratis (o ~USD 7/mes siempre activo).

Fase 1 · Probar en tu computador (10 min)

  1. Descarga el proyecto (GitHub → Code → Download ZIP, o git clone).
  2. En una terminal dentro de la carpeta, ejecuta npm start.
  3. Abre http://localhost:3000/ y las demos: /restaurante, /comida-rapida, /farmacia, /clinica, /cerrajero, y el panel /dashboard.
  4. Envía un formulario de prueba y verifica que aparece en el panel.

No necesitas npm install: el servidor no usa dependencias externas.

Fase 2 · Publicar online (Render)

  1. Entra a render.com (puedes iniciar con GitHub).
  2. New + → Web Service → conecta el repositorio woekstation.
  3. Start Command: node server.js. Build Command: vacío.
  4. Agrega las variables de entorno (Fase 3). Mínimo WEBHOOK_SECRET.
  5. Anota la URL que te da (la llamamos URL_PUBLICA).

Para probar sin publicar: npm start + npx ngrok http 3000 te da una URL_PUBLICA temporal.

Fase 3 · Notificaciones al dueño

Opción A (fácil): Telegram

  1. @BotFather → /newbot → copia el TOKEN.
  2. @userinfobot → copia tu Id.
  3. Variables: TELEGRAM_BOT_TOKEN y TELEGRAM_CHAT_ID.

Opción B: WhatsApp (Twilio)

TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_WHATSAPP_FROM, DUENO_WHATSAPP_TO (el del dueño del negocio).

Siempre: WEBHOOK_SECRET = un texto largo y secreto. Local con node --env-file=.env server.js.

Fase 4 · Montar la recepcionista IA (Vapi)

  1. vapi.ai → Assistants → Create Assistant.
  2. System Prompt: pega el guion según el rubro (carpeta agente-voz-ia/) y reemplaza los [CORCHETES].
  3. Model: OpenAI gpt-4o · Transcriber: Deepgram (es) · Voz en español.
  4. Tool / Function con su Server URL:
RubroHerramientaServer URL
Restaurante / comida / farmaciaregistrar_pedidoURL_PUBLICA/api/pedido
Clínicaagendar_citaURL_PUBLICA/api/cita
Oficios / urgenciasregistrar_urgenciaURL_PUBLICA/api/urgencia

En "secret" del tool pon el mismo WEBHOOK_SECRET. Agrega un parámetro negocio con el nombre del cliente.

Fase 5 · Conectar el número (Twilio)

  1. Compra un número en Twilio.
  2. En Vapi → Phone Numbers → impórtalo (SID + Auth Token) y asócialo al asistente.
  3. Cliente real: que mantenga su número y active el desvío de llamadas a la IA.

Fase 6 · Prueba del 100%

  • Llama al número y haz un pedido / cita / urgencia.
  • Llega la notificación al WhatsApp/Telegram del dueño.
  • Aparece en URL_PUBLICA/dashboard.

Fase 7 · Crear y entregar a una empresa cliente

  1. Copia una carpeta de cliente parecida (ej. public/restaurante).
  2. Renómbrala (ej. public/pizzeria-luigi).
  3. Edita solo su config.js: nombre, colores, teléfono/WhatsApp, catálogo y accion (pedido / cita / urgencia / lead).
  4. Pon el WhatsApp del dueño en su .env. Crea su asistente Vapi y conecta su número.
  5. Firma propuesta + contrato y cobra (carpeta documentos-negocio/).

Checklist final “está al 100%”

  • URL_PUBLICA carga.
  • WEBHOOK_SECRET igual en servidor y en el tool de Vapi.
  • Notificaciones llegan (Telegram o WhatsApp).
  • Asistente Vapi con el guion del rubro.
  • Tool apuntando a URL_PUBLICA/api/…
  • Número conectado.
  • Llamada de prueba → aviso recibido → aparece en el panel.
  • Propuesta y contrato listos.

Problemas comunes

SíntomaSolución
La IA contesta pero no llega nada al panelLa Server URL del tool debe ser URL_PUBLICA/api/… (no localhost) y el servidor activo.
Responde 401 / secreto inválidoEl "secret" del tool ≠ WEBHOOK_SECRET. Hazlos idénticos.
No llega WhatsApp/TelegramFaltan variables de entorno. Revísalas y reinicia.
La IA inventa preciosPega el menú completo dentro del System Prompt.
El sitio del cliente sale en blancoRevisa comas/comillas en su config.js y que se llame exactamente config.js.
